Read `SESSION.md` at the repo root, then give the user a tight briefing in this exact shape:

**Where we are**
One sentence naming the active milestone and its goal.

**Last session**
2–3 bullets of what got done, pulled from the most recent entry in the Session log.

**Ready to pick up**
The top 3 items from "Next up", each with a one-line pointer to what file / module they touch.

**Open questions**
Any items from "Open questions / parked decisions" that need a call this session.

Finish with: *"Which one first?"* — do not start any code until the user answers. Do not read any other files unless the user asks.
