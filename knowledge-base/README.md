Durable project memory. Research, benchmarks, and design decisions live here — one file per topic. Add liberally; this is the counterweight to short-lived chat context.

## Conventions

- One file per topic. Name files with a clear slug, not a date.
- Exception: *kickoff* / *session-XX* files that are snapshots of a moment in time get dated (e.g. `research-kickoff-2026-04-21.md`).
- Keep markdown clean: headings, tables, lists. Link between files liberally.
- When a decision changes, *update* the relevant file — don't leave stale guidance lying around.
- Cite sources with markdown links when external.
